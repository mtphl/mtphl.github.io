V580にCM14.1をインストール

published on September 17, 2017
はじめに ZTE Blade V580というAndroid端末のCustom Rom(CM14.1)をがんばって入れた話し。のーわらんてぃ。 Links Stock Rom LineageOS TWRP IMEI修正プログラム SuperSu LineageOSのsu TT175=V580である。OneTeamとかいうフォーラムでダウンロードしたいときは、適当にアカウント作って親指をクリックすればできるようになる。 SP Flash Tool MediatekのSoCを使っている端末で、Preloaderをふさいでいないものは、SP Flash Toolというプログラムを使うことで、任意のパーティションを上書きできるようだ。 使い方はDownload押したあとに、電源を切った状態の端末に、USBケーブルをぶっさすだけ。nvramはバックアップしておいた方がよいかも。モデム領域もリセットしたいときは、Format All + Dowloadとかをつかうべし。 IMEIがぶっこわれたら SP Flash Toolを使っていると、端末固有の領域をフォーマットしなければならない状況に落ちいるので、あらかじめIMEIを控えておいてIMEI修正プログラムのimei.cのこの char out_mask[12] = {0xAB, 0xA0, 0x6F, 0x2F, 0x1F, 0x1E, 0x9A, 0x45, 0x0, 0x0, 0x0, 0x0}; 部分を char out_mask[12] = {0x16, 0xE5, 0xF4, 0xD1, 0xE8, 0x6A, 0x42, 0x40, 0x0, 0x0, 0x0, 0x0}; と書きかえてコンパイルしたブツに、引き数として控えておいたIMEIを与えてやる。(MT6753ではマスクデータだけ変わってるだけみたい)するとMP0B_001_NEWとかいうファイルができるのでそいつをadbとかで、/data/nvram/md/NVRAM/NVD_IMEI/MP0B_001に上書きしてやる。(要root)ちゃんとBaseband versionとかがみえている状況なら、IMEIが復活するハズ。(みえていない状況でもこの操作でいろいろ直ることはある) 1.BootloaderのUnlock まず、TWRPを焼いていろいろしたいので、BootloaderをUnlockしやす。この機種は設定からOEMロック解除とかいう項目にチェック入れた後に、fastboot modeでfastboot oem unlockとしてやればよい。その後、端末がリセットされるので、一回再起動しておこう。 Read More...

Hugoの設定

published on May 20, 2016
やりたいこと Mathjaxを使っていい感じの数式を書きたいのだが、_(アンダースコア)がMarkdownで予約されているせいで、Latexの下付き文字と衝突して困る。 いちいちエスケープするのも、そのままtexにもっていけないのでいやだ。 しょうがないので、Hugoのソースコードをいじってやった。 Goのインストール Goの最新バージョンをインストールして、pathの設定もしておく。 Hugoのダウンロード 公式ページにあるように、 go get -u -v github.com/gohugoio/hugo しておく。 Blackfridayにpatch 調べたところHugoはMarkdownからhtmlへ変換するために、Blackfridayなるパッケージを使っているらしい。ちょうど、やりたかったことをやってくださった方がいたので、その成果を拝借した。 patchを”$GOPATH/src/github.com/russross/blackfriday”に”git apply”する。 mirror <- こっちが最新(2017/9/17) その他のパッケージもつじつまをあわせる。 patchを”$GOPATH/src/github.com/cpuguy83/go-md2man”に”git apply”する。 patchを”$GOPATH/src/github.com/spf13/hugo”に”git apply”する。 build cd $GOPATH/src/github.com/gohugoio/hugo go build -o hugo main.go もしくは cd $GOPATH/src/github.com/gohugoio/hugo go install siteの設定 config.tomlに以下の設定を追加する。 extensions = ["latexMath"] 問題点 現在のバージョンではなぜか”hugo server”がうまく働かない。 現在(2016/6/18)は動くようだ。 また、バージョンがあがればこんなことをしなくても、config.tomlに extensions = ["noIntraEmphasis"] を追加すれば動く…かもしれない。 Blackfridayでは対応しない線が濃厚です。 Commands go get -u -v github.com/gohugoio/hugo cd $GOPATH/src/github.com/russross/blackfriday wget http://www.mtphl.com/hugo_settings/blackfriday.patch git apply blackfriday.
